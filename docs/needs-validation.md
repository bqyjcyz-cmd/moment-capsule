# Moment Capsule — 需求真伪审判

## 结论：真需求，但需避开"日记 App"陷阱

### 反面证据

1. **日记类产品留存率极低**：情绪追踪 App 30天留存中位数仅 3.3%-6.1%；73% 尝试写日记的人最终放弃；Apple Journal 上线后被用户评价为"一周就忘了它的存在"
2. **地点类 App 结构性困境**：大多数人两点一线，地点变化低频；隐私顾虑高；GPS 精度问题
3. **"不知道写什么"是头号杀手**：不是"太麻烦"，而是"我有什么值得记的？"；断更产生焦虑后彻底放弃
4. **前人验证过但没做成功**：GeoLapse（2015）未商业化，Emotion Map 停留在 14 人试验

### 正面证据

1. **"给自己用"改变评估框架**：不需要百万用户留存率，只需自己持续使用。Day One 有用户连续记录 4000+ 天
2. **事件驱动而非时间驱动**：不是"每天要写"，而是"此刻有感触才记"。没有断更焦虑，三个月不记也正常
3. **极简记录已被验证**：Five Minute Journal 售出 100 万+ 份，核心原则是"门槛低到没有借口不做"
4. **市场空白真实存在**：无产品同时做到"强情感深度 + 强地点绑定"
5. **环境音是独特差异点**：几乎无竞品涉及，且是重现感受的强力媒介

### 致命陷阱：不要做成"日记 App"

| 日记 App 范式（危险） | 此刻胶囊应该是（安全） |
|----------------------|----------------------|
| 每天都该写点什么 | 有感触才记，没有就不记 |
| 打开 → 面对空白页 | 打开 → 自动定位 → 几句话+拍照 → 完 |
| 回看 = 翻阅列表 | 回看 = 地图重逢 + 沉浸氛围重建 |
| 价值在于"写"的过程 | 价值在于"未来重看"的那一刻 |
| 断更 = 失败感 | 三个月没记也完全正常 |

### 产品定位修正建议

**之前的定位**（有日记倾向）：
> "在某个地方驻足片刻，记录当下的感受"

**修正后的定位**（事件驱动+重逢导向）：
> "在有感触的时刻封存此刻，在未来的某天重新遇见"

核心差异：
- 强调"有感触的时刻"而非"在某个地方" → 去除记录义务感
- 强调"封存"而非"记录" → 暗示稀缺性和仪式感
- 强调"重新遇见"而非"回看" → 价值在未来而非当下

### 对 MVP 的具体影响

1. **去掉任何暗示"应该经常记录"的设计** — 不显示连续记录天数、不做记录频率统计、不发提醒鼓励记录
2. **强化"封存仪式感"** — 保存时的交互应该有一种"把瓶子扔进大海"的感觉，而非"又写了一篇日记"
3. **回看体验的优先级 ≥ 记录体验** — 回看时的氛围感是核心价值，记录只是手段
4. **空状态设计很重要** — 没有胶囊时不要显示"快去记录吧！"，而是"等待你的第一颗胶囊"

### 数据来源

- 情绪追踪 App 30天留存 3.3%-6.1%（JMIR 学术期刊，93款 App 研究）
- 全球 8% 人口坚持写日记，73% 尝试后放弃（ZipDo 统计）
- Apple Journal "一周就忘"（TechRadar 用户评测）
- Five Minute Journal 100万+ 销量（Female Startup Club 访谈）
- Day One 1500万下载，部分用户 4000+ 天连续记录
- 23% 的人第一周放弃新习惯，43% 一个月内放弃
