# Moment Capsule（此刻胶囊）— Product Review

## Summary

"此刻胶囊"是一个基于地理位置的个人情感记录工具。用户在某地驻足时用文字+照片（可选环境音）捕捉当下感受，未来通过地图或时间线沉浸式回看，重现当时的情感。技术路径为微信小程序+云开发先行。竞品调研显示该交叉领域无成熟商业产品，市场信号积极。

## Review Dimensions

### 1. Completeness

**缺失场景：**
- 胶囊的编辑和删除流程未定义 — 写错了/后悔了怎么办？
- 弱网/离线场景未覆盖 — 很多"驻足"的地方（山顶、地下室）可能没网
- 存储空间管理未考虑 — 照片+音频会快速消耗云存储额度
- 地点粒度未定义 — "西湖边"很大，相距50米的两个地方感受可能完全不同

**成功指标：**
- 三个指标都是主观感受，无法衡量。个人工具可以接受主观判断，但建议增加至少一个可观测的代理指标（如：回看停留时长、同一胶囊的重复打开次数）

**Scope 边界：**
- "关键词搜索"列入 MVP 是否必要？初期胶囊数量少，地图+时间线可能已够用

### 2. User Experience

**记录摩擦点：**
- 首次使用需连续授权定位+相机+麦克风，三个系统弹窗可能劝退
- 户外手机打字体验差，"1-2分钟完成记录"的预期需要验证
- 环境音录制的发现性和交互反馈未定义 — 用户怎么知道可以录？录制中怎么显示？

**回看体验定义不足：**
- "沉浸式回看"是核心差异点，但只给了方向没给底线 — 什么叫"足够沉浸"？
- 地图上胶囊密集/重叠时的展示策略未考虑
- 同一地点多次记录如何组织和呈现？

### 3. Technical Feasibility

> 无 TECH_CONTEXT.md，基于 brief 技术评估

**小程序沉浸体验天花板：**
- 微信小程序顶部导航栏和胶囊按钮无法隐藏，"照片铺满屏幕"的沉浸感会打折扣
- CSS 动画在小程序 WebView 中性能有限，"手写/淡入叠加"效果需降级方案
- **建议设计前先做技术原型，确认小程序内可达到的最佳沉浸效果**

**云开发存储成本：**
- 免费额度：存储 5GB、CDN 流量 5GB/月
- 每颗胶囊 ≈ 3-5MB（照片 2-5MB + 音频 ~500KB）
- 约 1000-1500 颗胶囊触顶，高频使用 1-2 年可能到达
- 需考虑图片压缩策略或付费升级计划

**离线能力：**
- 微信小程序可以用本地缓存暂存，联网后同步
- 但需要在架构层面提前设计离线队列

### 4. ROI

**开发量分布（估算）：**
- 记录流程：标准小程序能力，**中等工作量**
- 地图视图：微信地图组件+自定义 Marker，**中等工作量**
- 时间线视图：列表渲染，**低工作量**
- 沉浸式回看：**最大工作量**，也是核心差异点，值得重点投入
- 关键词搜索：云数据库全文搜索能力有限，投入产出比低

**80% 方案建议：**
- MVP 砍掉"关键词搜索"，用地图+时间线+时间筛选已够用
- 沉浸式回看先做基础版（照片全屏+文字覆盖+音频播放），不追求复杂动画

## Key Assumptions

| # | Assumption | Type | Risk | Validation Approach |
|---|-----------|------|------|---------------------|
| 1 | 用户在户外驻足时愿意花1-2分钟打字记录 | Usability | **High** | 自己使用2周，统计实际记录时长和完成率 |
| 2 | 照片+文字+环境音足以在回看时重现感受 | Desirability | **High** | 记录10颗胶囊后回看，主观评估感受重现度 |
| 3 | 小程序内的沉浸式回看体验能达到"足够好" | Feasibility | **Medium** | 设计阶段做技术原型验证 |
| 4 | 微信云开发免费额度够用1年以上 | Viability | **Medium** | 估算使用频率 × 单条体积，计算存储消耗曲线 |
| 5 | 个人使用不需要数据备份/导出 | Usability | **Low** | 云端数据丢失不可逆，建议预留导出能力 |

## Review Opinions

### Blockers (must resolve before design)

| # | Issue | Recommendation |
|---|-------|----------------|
| 1 | 沉浸式回看体验在小程序内的可行性未验证 | 设计前做一个技术原型：全屏照片+文字覆盖+音频播放，确认小程序能达到的效果底线 |
| 2 | 离线/弱网场景未覆盖 | 在 brief 中明确：MVP 是否支持离线记录？如不支持，需在 UX 上给出无网络时的明确提示 |

### Suggestions (improve but don't block)

| # | Issue | Recommendation |
|---|-------|----------------|
| 1 | MVP 包含"关键词搜索"，优先级可能过高 | 降为 P2，MVP 用地图+时间线+时间筛选替代 |
| 2 | 授权弹窗连续出现影响首次体验 | 设计一个引导页，解释为什么需要这些权限，分步授权而非一次性全部弹出 |
| 3 | 存储成本需要预案 | 照片压缩到 1MB 以内（小程序端压缩），延长免费额度使用寿命 |
| 4 | 同一地点多次记录的展示未定义 | 设计阶段需考虑：聚合为一组？按时间展开？叠加显示？ |
| 5 | 编辑/删除能力未提及 | 至少支持删除，编辑可选 |

### Future (noted, not for this iteration)

| # | Issue | Notes |
|---|-------|-------|
| 1 | 地理围栏重逢提醒 | 已明确排除至 App 阶段，合理 |
| 2 | 数据导出/备份 | 个人数据的长期安全性，建议 App 阶段实现 |
| 3 | 自动环境数据采集（天气、温度） | 低成本高回报的感受重现辅助信息，可在 V2 加入 |
| 4 | 从微信云迁移至自建后端 | 需在数据模型设计时预留迁移兼容性 |
