# Moment Capsule（此刻胶囊）— Product Brief

## Background & Motivation

当下的感受太容易被遗忘。快乐会褪色，痛苦会模糊，那些在某个街角、某家咖啡店、某个公园长椅上的真实感受，随着时间流逝变得面目不清。

这个产品的核心价值是：**在有感触的时刻封存此刻，在未来的某天重新遇见。** 不是看一条冰冷的记录，而是真的"回到那个瞬间"。

**关键定位：此刻胶囊不是日记 App。** 它是事件驱动的情感时间胶囊 —— 不要求每天写，有感触才记，三个月不记也完全正常。价值不在"记录了多少"，而在未来某天重新遇见时的那一刻。

## Target Users

- 自己（个人工具）
- 使用场景：在有感触的时刻驻足片刻（1-2 分钟），封存当下的感受
- 不是日常义务式的记录，而是有意识地在特别的瞬间停下来，和此刻相处

## Core Scenarios

| Scenario | User Action | Expected Result |
|----------|-------------|-----------------|
| 驻足记录 | 在某个地方停下来，打开小程序，写几句话、拍张照，可能录一段环境音 | 自动捕获位置，1-2 分钟内完成记录，形成一颗"胶囊" |
| 弱网/离线记录 | 在高空、山顶、地下室等无网络环境中记录 | 胶囊保存到本地，恢复网络后自动同步到云端，用户无需手动操作 |
| 地图回看 | 想回溯过去的感受，打开地图视图 | 看到散落在各处的胶囊，点开后沉浸式呈现：照片铺满、文字叠加、环境音播放 |
| 时间线回看 | 按时间顺序翻看过去的胶囊 | 像翻一本感受日记，每颗胶囊带有地点、时间、照片和文字 |
| 快速回溯 | 突然想找"那次在西湖边的感受" | 通过地图定位或时间筛选找到那颗胶囊 |
| 编辑/删除 | 想修改文字或删掉某颗胶囊 | 长按或进入详情后可编辑文字、删除整颗胶囊 |

## Solution Overview

**极简胶囊模式：**

- **记录流程**：打开 → 自动定位 → 写文字 + 拍照（主要）→ 可选拍一段短视频（≤15秒，替代 Live Photo）→ 可选录一段环境音 → 保存
- **回看方式**：
  - 地图视图：胶囊散落在地图上，点击进入沉浸式回看
  - 时间线视图：按时间顺序浏览
- **沉浸式回看**：照片为背景铺满屏幕，文字以手写/淡入方式叠加呈现，环境音自动播放 —— 目标是重建氛围而非展示数据。小程序阶段受限于顶部导航栏无法隐藏，需在设计前做技术原型验证可达到的效果底线
- **离线策略**：记录时先保存到本地（wx.setStorageSync + 本地临时文件），恢复网络后自动同步到云端。离线状态下定位使用最后一次缓存的位置或手动标记。用户无需感知在线/离线的差异
- **存储策略**：照片压缩到 1MB 以内，短视频 ≤15 秒（约 3-5MB），环境音 ≤60 秒（约 500KB）。单颗胶囊上限约 7MB（含所有素材），暂不限制总存储量，后续按需扩容
- **技术路径**：微信小程序 + 微信云开发（先行），后续迁移至独立 App

### 关于位置归属的决策

**胶囊的位置 = 封存时你所在的位置**，而非照片拍摄的位置。即使从相册选取旧照片，位置仍然是"此刻你在哪里"。原因：胶囊封存的是"此刻的感受"，位置是"此刻"的一部分。

### 权限拒绝的降级方案

| 权限 | 拒绝后的降级方案 |
|------|-----------------|
| **定位** | 允许用户手动在地图上标记位置，或标记为"未知位置"仍可封存 |
| 相机 | 仅可从相册选图，不阻断流程 |
| 麦克风 | 隐藏录音和录视频入口，不阻断流程 |

定位是核心能力，但**不能因为没有定位就完全阻断使用**。

## Information Architecture（页面结构）

```
TabBar（底部导航）
├── 地图（默认首页）    → 胶囊散落的地图视图，顶部有"历史上的今天"浮层提示
├── 封存（中间大按钮）  → 创建新胶囊的封存页
└── 时间线              → 按时间浏览胶囊，支持按年/月筛选

子页面
├── 沉浸式回看         → 从地图/时间线点击胶囊进入
├── 编辑胶囊           → 从回看页进入
├── 首次引导           → 仅首次打开展示（说明产品理念 + 分步授权）
└── 权限引导           → 某权限被拒后的说明页，引导去系统设置开启
```

## Tone of Voice（文案调性）

**温暖、克制、不打扰。** 像一个安静的朋友，不主动搭话，但你需要时它在。

- 不用感叹号，不用"赶快"、"马上"、"还等什么"
- 示例：
  - ❌ "快去创建你的第一颗胶囊吧！"  →  ✅ "这里会存放你的每一颗胶囊"
  - ❌ "保存成功！"  →  ✅ "已封存"
  - ❌ "你已经连续3天没记录了"  →  ✅ （不显示任何提醒）
  - ❌ "记录第28颗胶囊"  →  ✅ （不显示计数）

### 关于语音的定位

语音不是用来"说"的，而是用来"听"的 —— 记录的是周围的环境声音（咖啡店的嘈杂、雨声、街头音乐），而非自己的独白。这是一个重要的设计区分。

### 关于短视频的定位

短视频是 Live Photo 在小程序阶段的替代方案（微信小程序无法获取 Live Photo 完整数据）。3-5 秒短视频本质上是"加强版 Live Photo"：更长的动态画面+自带环境音。上限 15 秒，避免存储膨胀。后续 App 阶段可考虑支持原生 Live Photo。

## Design Principles（设计红线）

1. **不催促记录** — 不显示连续记录天数、不做记录频率统计、不发提醒鼓励记录
2. **封存仪式感** — 保存时的交互应该有"把瓶子扔进大海"的感觉，而非"又写了一篇日记"
3. **回看 ≥ 记录** — 回看时的氛围感是核心价值，记录只是手段
4. **空状态不施压** — 没有胶囊时不显示"快去记录吧！"，而是"等待你的第一颗胶囊"

## Success Metrics

1. **感受重现度**：翻看过去的胶囊时，那个感觉真的能回来（代理指标：回看时停留时长 > 30秒、同一胶囊是否被反复打开）
2. **回溯效率**：想找某个时刻的感受时，能快速定位到那颗胶囊
3. **使用意愿**：在有感触的时刻，愿意主动打开封存（而不是觉得麻烦）
4. **记录完成率**：打开封存页面后实际完成保存的比例 > 80%

## Tech Context Notes

> 当前无 TECH_CONTEXT.md，以下为初步技术评估

- **平台**：微信小程序（先行）→ 独立 App（后续）
- **后端**：微信云开发（云函数 + 云数据库 + 云存储），零运维
- **可用能力**：`wx.getLocation`（定位）、`wx.chooseMedia`（拍照/选图）、`wx.getRecorderManager`（录音）、地图组件
- **限制**：小程序无法后台持续定位，地理围栏能力弱，"重逢提醒"需到 App 阶段实现
- **风险**：沉浸式回看体验在小程序内的表现力有限（无法全屏沉浸、动画能力受限），需在设计阶段评估可行性
